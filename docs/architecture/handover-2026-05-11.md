# Handover — Per-state event identity + government as primary anchor (2026-05-11)

**Author**: agent session ending 2026-05-11 (after the 8-state ECI ingest commit `29da524`).
**Audience**: the next session (or contributor) who picks up the IA-reset thread.
**Companion docs**: [ADR-0023](decisions/0023-election-event-identity-per-place.md), [docs/concepts/government-vs-election.md](../concepts/government-vs-election.md), [TODO/IA-RESET-PLACE-FIRST-WITH-TOPIC-FRONT-DOOR.md](../../TODO/IA-RESET-PLACE-FIRST-WITH-TOPIC-FRONT-DOOR.md) §P2.5.

## What problem was solved

A citizen typing `/s/andhra-pradesh` (or any state outside the May-2026 cohort: AP, AR, BR, HR, JK, JH, MH, OD, SK, DL) saw an unrecoverable 404 because seven separate route components had hardcoded `const event = "AcGenMay2026"` and asked for `/data/elections/AcGenMay2026/S01/result.summary.json`. AP has never had a May-2026 election.

Root cause was structural, not a typo: the IA carried a global "Election" dropdown (from when AcGenMay2026 was the only ingested event) that encoded the federal falsehood that elections are nationally synchronous. India does not have a "current election"; each state has its own cycle (AP polled in 2024, Bihar in 2025, J&K in 2024 after six years of President's Rule, etc.).

Two subagents (Architect Gregor + Governance Strategist) consulted in parallel converged on the same diagnosis: kill the dropdown, make event identity per-place, and promote government-timeline (continuing condition) above election (discrete event) as the citizen's primary anchor. ADR-0023 is the canonical record.

## Lessons (worth memorising)

- **The Architect was right that subagent runs in parallel save real time** — Gregor + Strategist consultations done concurrently delivered convergent diagnoses in ~5 minutes. Subagent state-name guesses for ECI codes (S07/S13/S18/S27) were WRONG (gave Goa/HP/MP/UP instead of Haryana/Maharashtra/Odisha/Jharkhand); always cross-check against `events.py`.
- **A "graceful 404" is not a fix when the underlying control is conceptually wrong.** Skipping the B0 hotfix and going straight to B1+B2 saved a throwaway codepath.
- **Hand-authored data is not "stale by definition"** if it's small, citizen-meaningful, and validated by CI against a generated source. The `election-events.json` ↔ `events.py` consistency test makes the duplication safe.
- **Frozen-in-time principle**: government timelines are easier to ship correct (4 states done) than 10 more in a panic. Honest "coming soon" beats invented data.
- **"AcGenMay2026" must never appear in citizen-facing chrome** — the `display` field in `election-events.json` is the only label. Codes appear in URL path segments only when explicit (`/s/<state>/elections/<event_id>`), never in chrome strings.

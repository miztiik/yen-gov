## 1. Indicators — what's loaded

> **Auto-generated tables live at [data-inventory.md § 1](data-inventory.md#1-indicators).** Each indicator artifact is listed with its unit, time grain, span, row count, entity count, source host, and a 7-cell **Temporal Richness** meter (`● ● ● ○ ○ ○ ○ 3/7`). Re-run `python -m yen_gov coverage` after every ingest to refresh.

How to read the meter (full rationale in [docs/architecture/backend/coverage.md](../architecture/backend/coverage.md)):

- 7 cells × 3 fiscal years each, FY06–FY26. Rightmost cell is newest.
- A cell is `●` if the indicator has at least one data point inside that bucket.
- Snapshots (single-month CEA series, ad-hoc geodata) carry a `(snapshot)` tag.

### What the categories tell you

Per-indicator definition, methodology vintage, series breaks, denominator, exclusions, sources, and citation live inside the artifact JSON under `datasets/indicators/in/<topic>/<id>.json` (github-browsable; the single source of truth per CLAUDE.md §5). The auto-generated `docs/reference/indicators/` markdown tree was retired 2026-05-17 — see [`docs/reference/indicators/README.md`](indicators/README.md) and [TODO/20260517-coverage-temporal-range-plan.md](../../TODO/20260517-coverage-temporal-range-plan.md) Phase #4a. This section keeps only the **cross-cutting story** of what each topic is for and where it's deep vs shallow. For the row-level shape table (`unit`, `time grain`, `span`, `Temporal Richness`), see [data-inventory.md § 1](data-inventory.md#1-indicators).

- **Energy (ICED + CEA).** ICED gives long annual per-state series (~10y, FY15–FY25); CEA gives a current-month nameplate-capacity snapshot per fuel. They overlap on installed capacity — ICED is the long-series, CEA is the freshness check. Tier-1/2/3 ICED ingest (2026-05-15) added 8 new endpoints across generation mix, plant-load-factor, CO₂, DISCOM efficiency, RPO compliance, coal/oil consumption, and procurement mix; ~86 ICED endpoints (climate, AQI, water, transport demand, captive, sub-source generation) remain unbound. See [data-inventory.md § 1](data-inventory.md#1-indicators) for the full list.
- **Fiscal (RBI + data.gov.in OGD).** Two distinct shapes: **national long series** (~8 indicators, RBI Handbook / State Finances, FY08–FY26, 7/7 buckets — the canonical "all-states-combined" view) and **per-state series** (`outstanding_debt_pct_gsdp` 31 states FY08–FY26 is the only multi-decade per-state fiscal series; `state_pension_expenditure_inr_crore` 30 states FY05–FY25 ingested 2026-05-16; transfers split across two non-overlapping windows — see the mis-framing-debt row in §5d).
- **Economy + Demography (ICED + RBI Handbook).** ICED ships the `stateWiseDeepDive` bundle (GSDP / GVA / per-capita NSDP, single-base 2011-12, ~FY11–FY24) plus 5 socio-economic siblings ingested 2026-05-15 (per-capita NSDP constant; per-capita private consumption; population-by-sex Census frame; HDI snapshots 2011/2017; national GHG by IPCC sector). RBI Handbook 2024-25 ingest (2026-05-15) added the 4 vintage-spliced **deep-history** NSDP / per-capita-NSDP siblings going back to FY 1994–95, with `series_breaks[]` declaring the four MoSPI rebase boundaries (1993-94 / 1999-2000 / 2004-05 / 2011-12) and `vintage` per row. Pair the spliced and single-base siblings depending on whether you need depth or single-base purity.
- **Prices / inflation (RBI Handbooks, 2026-05-16).** 7 artifacts — 3 national index series (`national_wpi_all_commodities_index_annual` 51y FY75–FY25 spliced across 5 WPI bases; `national_cpi_iw_index_annual` 32y FY94–FY25; `national_cpi_combined_index_annual` 14y FY11–FY25, the RBI's monetary-policy anchor) plus 4 state CPI sub-index inflation rates (general / food / fuel / urban-housing) 34 states × 11y. All `direction: "neutral"` per Hans's framing — inflation is intrinsically directionless; what matters is direction-of-change vs the 4% target. National series ship index *level* (rebases declared as `definition_change` breaks); state series ship YoY %.
- **Health / vital statistics (SRS via RBI Handbook, 2026-05-16).** 5 artifacts — Crude Birth Rate, Crude Death Rate, IMR, TFR (calendar-year, 35 states × ~20y; SRS Office of the Registrar General); plus state public health expenditure (fiscal-year, 34 states × 8y). IMR is `lower_is_better`; the others are `neutral` (CDR is age-confounded, CBR/TFR drop naturally with development). MMR + Life Expectancy + Doctors-at-PHCs deferred from this batch on shape grounds (multi-year SRS windows, multi-sheet workbooks).

### 6d. New components needed

- **`SeriesBreakAnnotation.svelte`** — vertical dashed rule + small caret on any time axis (line, sparkline, year-slider tick). Mandatory for the long-history series.
- **`SnapshotBadge.svelte`** — extracted from the choropleth's stale-chip; reused by Ranked + SmallMultiples so the CEA snapshot is consistently flagged across all three lenses.
- **`DirectionLegendCue`** — one-word verbal tag in legend ("lower is better", "neutral · darker = higher", "higher is better"); ~5 lines of Tailwind, eliminates the colour-only signalling smell.
- **`VintageTooltipLine`** — small addition to [`ChartTooltip.svelte`](../../frontend/src/lib/ChartTooltip.svelte) so per-row `vintage` (e.g. "Base 2011-12" / "First Advance Estimates") shows on hover, not only as a footer line.
- **`RebaseBanner.svelte`** — small amber strip on the choropleth when the year slider lands on or past a `series_breaks` boundary.

## 7. How to refresh / regenerate

| Want to refresh… | Run |
| ---------------- | --- |
| Validate every dataset against its schema | `python -m yen_gov validate` (from `backend/` with the venv) |
| Regenerate the auto election-coverage report ([data-inventory.md](data-inventory.md)) | `python -m yen_gov coverage` |
| Run the full backend test suite | `cd backend && pytest -q` (the two LGD tests above will fail until that pipeline lands; `--deselect` if needed) |
| Run the frontend contract tests (every dataset) | `cd frontend && npm test -- src/contracts/datasets-conform.test.ts` (9,312 tests) |
| Ingest a new ECI election | Add the event id to `config/processing.json`, then `python -m yen_gov pipeline run --event <EventSlug>` |
| Refresh ICED state series | Re-run the ICED adapter; payload is fetched live and decrypted in-process |
| Refresh CEA installed capacity | Operator: `Invoke-WebRequest` on the next monthly XLSX into `.runtime/raw/cea/`, then run the CEA adapter |

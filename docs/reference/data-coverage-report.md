## 1. Indicators — what's loaded

> **Auto-generated tables live at [data-inventory.md § 1](data-inventory.md#1-indicators).** Each indicator artifact is listed with its unit, time grain, span, row count, entity count, source host, and a 7-cell **Temporal Richness** meter (`● ● ● ○ ○ ○ ○ 3/7`). Re-run `python -m yen_gov coverage` after every ingest to refresh.

How to read the meter (full rationale in [docs/architecture/backend/coverage.md](../architecture/backend/coverage.md)):

- 7 cells × 3 fiscal years each, FY06–FY26. Rightmost cell is newest.
- A cell is `●` if the indicator has at least one data point inside that bucket.
- Snapshots (single-month CEA series, ad-hoc geodata) carry a `(snapshot)` tag.

### What the categories tell you

- **Energy (ICED + CEA).** ICED gives 8 per-state energy series, all states/UTs, ~10 years annual (most show `○ ○ ○ ● ● ● ● 4/7`). CEA gives a current-month per-state nameplate-capacity snapshot by fuel (`○ ○ ○ ○ ○ ○ ● 1/7 (snapshot)`). They overlap on installed capacity; ICED is the long-series, CEA is the freshness check. ICED also publishes two national long series (`national_primary_energy_supply_mtoe`, `national_final_energy_consumption_by_sector_mtoe`) that fill all 7 buckets.
- **Fiscal (RBI + data.gov.in OGD).**
  - **National long series (8 indicators)** — RBI Handbook / State Finances publication, FY08–FY26, all 7 buckets full. The canonical "all-states-combined" view.
  - **State debt** — `outstanding_debt_pct_gsdp` is per-state, 31 states, FY08–FY26 — the only multi-decade per-state fiscal series we ship (`● ● ● ● ● ● ● 7/7`).
  - **State transfers — two slices**: `net_transfers_from_centre` (RBI, 31 states, FY24–FY26 only, `2/7`) and `centre_transfers_gross` (data.gov.in OGD, 28 states, FY17–FY23, `3/7`). Together they cover FY17–FY26 with a gap shape; neither alone is complete.
- **Economy + Demography (ICED).** Five series bundled in the ICED `stateWiseDeepDive` payload (one HTTP fetch covers the whole NITI energy/economy/demography wedge). GSDP / GVA show `4/7`; per-capita NSDP series go back to FY05 and reach `6/7`.
- **Economy — RBI Handbook spliced long series (new, 2026-05-15).** Four artifacts ingested directly from the RBI Handbook of Statistics on Indian Economy 2024-25 edition (Tables 5, 6, 9, 10):
  - `economy/state_nsdp_current_inr_crore` — 33 entities × 31y (FY 1994–95 → FY 2024–25), `7/7`.
  - `economy/state_nsdp_constant_inr_crore` — same shape.
  - `economy/state_per_capita_nsdp_current_inr_long` — 34 entities (incl. all-India NNI reference) × 25y (FY 2000–01 → FY 2024–25), `7/7`.
  - `economy/state_per_capita_nsdp_constant_inr_long` — same shape.

  Each row carries a `vintage` field naming the MoSPI base year that produced the value (`Base 1993-94` / `Base 1999-2000` / `Base 2004-05` / `Base 2011-12`); for any year present under multiple bases, the most recent base wins. `series_breaks[]` declares the four rebase points and the FY 2011–12 definition change (NSDP at factor cost → NSDP at basic prices). The renderer must refuse to compute growth rates that span a `definition_change` break, and should label `coverage_change` rows for Telangana (back-projected pre-2014-15) and J&K (UT-only post-2019-20). Pair these with the existing single-base ICED siblings (`state_per_capita_nsdp_current_inr`, `_constant_2011_12_inr`) when single-base purity matters.

- **Prices / inflation — RBI Handbook (new, 2026-05-16).** Seven artifacts ingested from the RBI Handbook of Statistics on Indian Economy 2024-25 (Tables 36, 37) and the RBI Handbook of Statistics on Indian States 2024-25 (Tables 108–111). All `direction: "neutral"` per Governance Strategist guidance — inflation is neither intrinsically good nor bad, the citizen-relevant question is direction-of-change vs the RBI's 4% target.
  - **National — deep history.** `prices/national_wpi_all_commodities_index_annual` (51y, FY75–FY25, spliced across 5 WPI base years 1970-71 / 1981-82 / 1993-94 / 2004-05 / 2011-12); `prices/national_cpi_iw_index_annual` (32y, FY94–FY25, CPI for Industrial Workers — the deepest CPI India publishes); `prices/national_cpi_combined_index_annual` (14y, FY11–FY25, RBI's official monetary-policy anchor since 2014). Each ships index *level* (not YoY %); convert to inflation in the renderer. `series_breaks[]` declares each rebase as a `definition_change` so the renderer must not compute growth across them.
  - **State — last decade.** `prices/state_cpi_general_inflation_pct`, `_food_inflation_pct`, `_fuel_inflation_pct`, `_housing_urban_inflation_pct` — each 34 entities × 11y (FY15–FY25), `value_kind: "rate"` (RBI publishes already as YoY %, no further computation). Housing is urban-only by NSO methodology, not a yen-gov omission. Together they expose the citizen-relevant decomposition: when headline inflation moves, a citizen can see whether it was food (supply shock), fuel (subsidy / global crude), or housing (rent revision).

- **Fiscal — state pension expenditure (new, 2026-05-16).** `fiscal/state_pension_expenditure_inr_crore` from RBI Handbook of Statistics on Indian States 2024-25 Table 171: 30 states × 21 fiscal years (FY05–FY25), ₹ Crore. Captures retirement and family pensions paid to state-government employees (pre-NPS hires) on the revenue account; does NOT cover IGNOAPS / IGNWPS social pensions or NPS contribution flows. Rapidly-growing line item in most state budgets — directly relevant to the Old Pension Scheme (OPS) restoration debate. Latest 3 years carry RBI's `(A)` / `(RE)` / `(BE)` revision-tier suffix on the underlying source; suffixes are stripped at parse time but the FY24/FY25 values should be read as Revised / Budget Estimates rather than Actuals.

- **Health / vital statistics — SRS via RBI Handbook (new, 2026-05-16).** Five artifacts from the RBI Handbook of Statistics on Indian States 2024-25 (Tables 2, 3, 4, 6, 18), ultimately sourced from the Sample Registration System (Office of the Registrar General of India). Calendar-year (not fiscal) for the SRS series:
  - `health/state_birth_rate_per_1000` — Crude Birth Rate, 35 entities × 20y (CY04–CY23), `direction: "neutral"`.
  - `health/state_death_rate_per_1000` — Crude Death Rate, 35 entities × 20y, `direction: "neutral"` (crude rate is age-uncontrolled — ageing states like Kerala will read structurally higher).
  - `health/state_infant_mortality_rate_per_1000` — IMR, 35 entities × 20y, `direction: "lower_is_better"` (the most cited single summary of state public-health performance).
  - `health/state_total_fertility_rate` — TFR, 22 entities × 21y (CY03–CY23). Replacement level 2.1; India crossed below replacement around 2020.
  - `health/state_public_health_expenditure_inr_crore` — fiscal-year, 34 entities × 8y (FY13–FY20), ₹ Crore (state revenue + capital health heads). Direction is `neutral` because spending is an INPUT not an OUTCOME — pair with IMR / Life Expectancy for value-for-money analysis.

  Deferred from this batch (different shapes need careful handling): T05 Maternal Mortality Ratio (multi-year SRS windows, no per-year row), T07 Life Expectancy at Birth (5-sheet workbook with overlapping period windows × Male/Female/Total), T16 Doctors / Specialists at PHCs / CHCs (multi-section workbook), T17 Government Hospitals & Beds (Rural/Urban/Total breakdown across two reference dates). All cached locally under `.runtime/raw/rbi/handbook_states_2024_25/`; see `tools/rbi_hbs_ingest_inflation_pension_health.py` for the ingest pattern to follow when these land.

## 7. How to refresh / regenerate

| Want to refresh… | Run |
| ---------------- | --- |
| Validate every dataset against its schema | `python -m yen_gov validate` (from `backend/` with the venv) |
| Regenerate the auto election-coverage report ([data-inventory.md](data-inventory.md)) | `python -m yen_gov coverage` |
| Run the full backend test suite | `cd backend && pytest -q` (the two LGD tests above will fail until that pipeline lands; `--deselect` if needed) |
| Run the frontend contract tests (every dataset) | `cd frontend && npm test -- src/contracts/datasets-conform.test.ts` (9,312 tests) |
| Ingest a new ECI election | Add the event id to `config/processing.json`, then `python -m yen_gov pipeline run --event <EventSlug>` |
| Refresh ICED state series | Re-run the ICED adapter; payload is fetched live and decrypted in-process |
| Refresh CEA installed capacity | Operator: `Invoke-WebRequest` on the next monthly XLSX into `.runtime/raw/cea/`, then run the CEA adapter |

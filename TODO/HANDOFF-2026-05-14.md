## 5. How to drive the RBI HBS-IS workflow (for any of A, B, partial D)

This is the recipe the session converged on — follow it verbatim:

1. **Download** via `python tools/rbi_hbs_download.py <out_dir> <referer> <url> <name>`. **Never** use raw `Invoke-WebRequest` — RBI's CDN rejects non-browser UAs and returns HTML error pages that fail downstream as `BadZipFile`.
   - `out_dir` = `.runtime/raw/rbi/handbook_states_2024_25` (gitignored — this is intermediate per ADR-0003).
   - `referer` = `https://www.rbi.org.in/Scripts/AnnualPublications.aspx?head=Handbook+of+Statistics+on+Indian+States`.
2. **Recon** the workbook layout: `python -c "import openpyxl; wb=openpyxl.load_workbook(r'<path>', data_only=True, read_only=True); [print(s,'=>',list(wb[s].iter_rows(values_only=True))[:8]) for s in wb.sheetnames]"`. **Do this BEFORE writing the spec.** RBI Power had two layout patterns; assume nothing.
3. **Extend** `tools/rbi_hbs_ingest_power.py` (or fork to `tools/rbi_hbs_ingest_health.py` if section is meaningfully different — it shouldn't be). Add a SPECS entry per artifact. The existing parsers cover Pattern A (states × FY) and Pattern C (multi-period grouped). For calendar-year data, pass `calendar=True` to `parse_state_year_table`. For new layouts, write a new helper rather than overload existing parsers.
4. **Bump** `FETCHED_AT` to current UTC.
5. **Emit & validate**: `python tools/rbi_hbs_ingest_power.py` then `cd backend; python -m pytest -x -q tests/test_validate.py tests/test_datasets_integrity.py` (~100s).
6. **Refresh inventory**: `cd ..; python -m yen_gov coverage`.
7. **Commit on a feature branch** (`feat/rbi-hbs-<section>-<ids>`), explicit `git add <each-file>` (never `git add .` per CLAUDE.md §8), commit message via `git commit -F .tmp_commit_msg.txt` (multi-line `-m` hangs on Windows PowerShell).
8. **Verify** before push: `git diff-tree --no-commit-id --name-status -r HEAD` — see lesson in §2 above.
9. **Merge** to main with `--no-ff`, push both branch and main.
